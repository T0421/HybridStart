关于

概述

HybridStart 是一套混合应用开发框架，提供丰富的UI组件、清晰的代码管理、简明开发模式，底层基于apicloud，可以替换成任意提供多webview交互的混合应用引擎(例如appcan)。

目录结构

           
  |-- docs/                 //文档(不需要上传打包平台)
  |-- error/                //app错误页
  |-- res/                  //app静态资源(图片、模板、前端插件)
  |-- sdk/                  //框架文件
  |    |-- modules/             //封装功能模块
  |    |-- font/                //app字体图标
  |    |-- core.js              //核心库
  |    |-- server.js            //app业务方法
  |    |-- common.js            //app页面公用代码
  |    `-- ui.css               //app UI
  |-- view/                 //app页面
  |-- config.js             //框架配置
  `-- config.xml            //apicloud配置

代码组织

HybridStart 项目的代码整体被分成SDK+View两部分。

SDK包含UI、js核心库、常用功能封装、公用代码。js核心库core.js集成seajs、jQuery2、etpl(模板引擎)、xss(xss防御)，以及app对象(对混合应用引擎的封装)，除了seajs和app对象以外都是非必须；modules/内是常用功能模块，包括上传、下载、提示框等；server.js和common.js都是业务代码中提取的共用方法或代码。

View包含所有的app页面，每个页面都体现为一个文件夹，其中至少包含temp.html、style.css、script.js三个文件，页面文件夹都放在view/中，每个页面都引用SDK提供ui和方法来简化开发，公共方法也便于管理。

开发模式

openView

多webView切换机制，解决了混合应用页面切换不流畅的老大难问题，使app开发可以像web开发一样通过“链接”跳转“页面”，但这种机制在app开发中存在一些先天缺陷，比如状态维护、参数传递；也带来了新的问题，比如多页面内存占用，使刚接触的人无从下手。

HybridStart 基于引擎功能封装了openView方法，可以方便的跳转到指定页面，并解决传参、关闭后台等一系列问题。

生命周期

app的生命周期为 root -> index <--> page | 返回-> index -> exit，启动页(view/root)通常没有任何展示内容，只执行状态检测、插件初始化、监听、数据预取等初始化操作，完成后直接跳转到首页(index)，首页与展示页之间任意跳转返回，但是在首页拦截返回键，使得无法继续返回启动页，而是退出app，整个生命周期内启动页都在后台运行。

交互约定

约定前后端交互数据基本格式如下：

          
{
    "status": "Y",      //请求的状态 "Y"/"N"，也可以根据情况扩展其他
    "data": [{...}],    //请求的数据 数组或对象
    "msg": "",          //【可选】服务端提示信息
    "count": [number]   //【可选】当获取列表数据时，需附加count数据指明列表总数，用于前段判断是否还有更多
}

快速开始

  1. 在apicloud平台创建一个项目，选择以下模块：UIMultiSelector,UIActionSelector,ajpush,bMap,photoBrowser,dialogBox,push,mam
  2. 将代码检出到本地，备份config.xml并清空其他文件，然后将本项目除了docs/以外所有文件拷贝进来，改一下config.xml里的appid之类的关键配置
  3. 上传，平台编译

app.openView(param, leval1, [leval2])

跳转页面方法，leval1和leval2分别代表view/内的文件夹名称，openView方法将打开目标目录中的temp.html页面，因此view/内部目录最多支持两级，页面文件夹内模板文件必须是temp.html，详细参数：

param

[object/string]openView方法给新页面传值的方式是先将数据存在本地存储中的crossParam键中，新页面打开后可以通过app.ls.val("crossParam")获取。

当param为非object类型时，将直接被存入crossParam，若为object类型可以传入更多配置，其中param.param为新页面参数，若不需要传参数和修改任何配置，则要传入null，此时将清空之前的crossParam值，避免页面参数混淆
param.param

作为本地存储crossParam的值，供新页面使用
param.duration

页面切换动画时长，默认appcfg.set.animateDuration
param.anim

页面切换动画时长，默认appcfg.set.windowAnimate
param.fullPath

默认false，若为ture，将参数leval1作为页面完整地址访问，并忽略leval2的值
param.pop

是否以弹层形式在当前窗口打开，默认false

默认false，若为true，还可以传param.top和param.height，指示弹层的顶部位置和高度，默认都是0
param.bar

是否以带标题栏的方式打开，默认false

默认false，若为true，还可以传param.title作为标题栏内容
param.closeself

默认false，跳转到新页面后是否关闭当前页面
param.closeback

默认false，跳转到新页面后是否关闭所有后台页面
leval1

目标页面在view/内的一级目录，如：

                        
//将打开 "view/news/temp.html"
app.openView(null,'news');

leval1也可以是url，用于打开web页面，此时leval2将被忽略
leval2

目标页面在view/内的二级目录，如：

                        
//将打开 "view/news/detail/temp.html"
app.openView(null,'news','detail');

  var loginForm = $('#reg_form').Validform({
      beforeCheck:function(form){
        //在这做验证前操作，可以通过"return false"语句中断验证
      },
      callback:function(res){
        
      }
    });
beforeSubmit

验证后、提交前的操作，例如要在表单验证之后提交之前，先对某些表单数据做合并或格式转化操作，应该在"beforeSubmit"回调中做处理，如果该回调中明确 "return false" 的话,表单将不被提交：

                      
  var loginForm = $('#reg_form').Validform({
      beforeSubmit:function(form){
        //在这里对表单数组做处理，也可以通过"return false"语句中断提交
      },
      callback:function(res){
        
      }
    });
